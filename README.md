# pretrain_plus_SFT
Пайплайны для обучения LLM: pretrain &amp; SFT

## Описание проекта

В проекте реализован полный пайплайн обучения **LLM**:

1. сбор и очистка текстового корпуса
2. обучение собственного BPE-токенизатора
3. **pretraining трансформерной модели с нуля**
4. генерация текста для оценки качества
5. SFT на датасете
6. дообучение модели с использованием **LoRA**
---

# Этап 1. Pretraining

## Данные

Для претрейна используется корпус русской художественной литературы. 
Ссылка на датасет: https://github.com/JoannaBy/RussianNovels/tree/master/corpus

---

## Препроцессинг данных

Перед обучением выполнена очистка данных:

- удаление предложений с латиницей
- нормализация пунктуации
- разбиение на чанки (предложения) фиксированной длины. параметры: context_length = 512, chunk_size ≈ 1400
- удаление дубликатов

После очистки текст объединяется в единый корпус.

---

## Создание и обучение собственного токенизатора

Обучен BPE-токенизатор (vocab_size = 3000).

Использованы спец. токены <bos> <eos> <unk> <pad>

Использование собственного токенизатора позволяет:

- лучше адаптироваться к русскому языку
- эффективно кодировать редкие слова через подслова
- повысить эффективность обучения

---

## Архитектура модели

Используется трансформерная архитектура LlamaForCausalLM (~132М параметров):

- hidden_size = 1024
- num_hidden_layers = 16
- num_attention_heads = 16
- num_key_value_heads = 8
- intermediate_size = 1536

---

## Обучение

В процессе обучения используется:

- генерация текста на валидации
- callback для отслеживания качества
- ранняя остановка обучения

Финальные значения ф-ций потерь:
- train loss ≈ 2.94
- validation loss ≈ 3.83

---

## Генерация текста

Для оценки качества генерации используются тестовые промпты.

Модель генерирует продолжение текста на основе обученного языкового распределения.

При генерации мы получаем нормальный и ожидаемый результат для pretrain. Ввиду маленького размера словаря BPE, модель вынуждена оперировать слогами, что может приводить к появлению "артефактов". Когда модель не уверена, она начинает перебирать наиболее вероятные слоги. Однако, модель хорошо выучила пунктуацию, стиль обращений, что означает, что модель научилась структуре языка. Это главное требование, которого мы добивались в pretrain-стадии обучения модели

---

# Этап 2. Post-train SFT

## Датасет 

Используется русскоязычный инструктивный датасет в диалоговом формате. 

Ссылка на датасет: https://huggingface.co/datasets/d0rj/alpaca-cleaned-ru

---

## Подготовка данных

Выполнена очистка и подготовка:

- разделение на train / validation / test
- удаление пересечений между выборками
- дедупликация примеров
- удаление повторяющихся диалогов

Это снижает риск утечки данных между выборками

---

## Дообучение модели

Используется базовая модель Qwen2.5-0.5B, т.к. для SFT-этапа можно использовать значительно меньше данных.

Дообучение выполнено с помощью LoRA (Low-Rank Adaptation), с параметрами:

- r = 8
- alpha = 16
- dropout = 0.05

Обучаются только адаптационные веса, что значительно уменьшает количество обучаемых параметров.

Параметры обучения:

- epochs = 3
- batch_size = 2
- gradient accumulation
- fp16 training

Используется SFTTrainer (TRL).

---

## Оценка адекватности генерации на финальных данных

После SFT модель тестируется на русскоязычных вопросах.

Примеры:

- сколько планет в солнечной системе?
- расскажи стих

Модель генерирует осмысленные ответы на русском языке и демонстрирует способность поддерживать диалог.
